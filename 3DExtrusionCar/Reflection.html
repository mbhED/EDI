3D Extrusion Car Reflection

For this project we modeled and 3D printed a toy car. The goal of the project was to accurately represent the toy car. We worked in X, Y and Z planes to create the final 3D form.

Guidelines

  • Car designed as an intersection of three forms
  • Design car in a 1:1 ratio
  • Size accuracy Â±1%
  • Design is exported as stl
  • Student orients model in Cura for 3D printing to minimize support material
  • File is exported as a gcode
  • Final design is shared out as an embedded STL on an HTML webpage

Project Steps

Drawing:

  • Draw car to scale as an Engineering Drawing
  • Scan/photograph drawing
  • Capture screenshots of each plane side

3D Modeling:

  • Draw each profile on each plane side using your engineering drawing
  • Use the extrusion tool to remove the negative spaces from the 3D form
  • Export as an STL file

Slicing and Printing:

  • Import design into Cura to make any final scaling changes
  • Position 3D model on build plate to minimize support martial
  • Set nozzle to 0.6 mm
  • Export as a GCODE file
  • Print model using the 3D printer

Documentation:

  • Create CAD drawing with margin of error analysis
  • Import design to Modelo to make a 3D model
  • Create a reflection using an HTML document with blurb and 3D render

Setbacks and Solutions

One setback occurred while creating my reflection. I included the url of the image but it had the wrong tag so it did not load on the website. To fix this I re-uploaded the pictures onto Github, making sure they were jpgs and not pdfs.

Another setback I encountered was that my engineering drawing did not fit correctly inside my box at the beginning of the 3D Modeling process. The X and Y planes were much shorter than the length of the box, and the Z plane was too thin.

To fix this problem, I only traced half the profile of the Z plane in Onshape and mirrored it over the center to make it the correct width. However, I could not mirror the other two planes because the sides were not symmetrical. My lines on my engineering drawing were too thick, so I did not know exactly where on the line to trace in Onshape. Using a ruler and only drawing one line instead of making the lines dark and going over them would have reduced the margin of error because the Engineering Drawing would have been more accurate.
